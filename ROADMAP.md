# ROADMAP — Math Expression Bugs

- [x] paren-chain в `if()`: `if ((a+b)*c > 15)` — `constructcompare()`
- [x] paren-chain в `return`: `return (x+y)*(x-y)`
- [x] paren-chain в аргументах функций: `foo((b+c)*d)`
- [x] вложенные скобки в присваивании: `(a+b)*(c+d)`
- [x] приоритет `*` над `+`: `a + b * c + d`
- [x] `//` комментарии с LF окончаниями строк

## ❌ Known Bugs

### 1. while/do-while paren-chain
`while ((a+2)*3>5)` и `do {} while ((a+2)*3>5)` — падают.
- **Причина**: `dowhilefast()` не отслеживает глубину скобок в первом проходе. `SkipParam()` останавливается на внутренней `)`.
- **Файл**: `tokc.cpp:4214`
- **Сложность**: средняя

### 2. Приоритет операторов с вызовами функций
`fn(a) + fn(b) * c` компилируется как `(fn(a) + fn(b)) * c`, а должно быть `fn(a) + (fn(b) * c)`.
- **Причина**: вызов функции внутри `do_e_axmath2()` не сохраняет промежуточные результаты для последующих операторов.
- **Файл**: `tokc.cpp` (функции `do_e_axmath2`, `constructcompare`)
- **Сложность**: высокая

### 3. Порча состояния компилятора
После любого присваивания вида `x = fn(a) + b * c` (функция + операторы), следующий за ним `if()` с paren-chain падает с `')' expected`.
- **Причина**: вероятно, не восстанавливается какой-то глобальный флаг/указатель после `do_e_axmath2()` при наличии вызова функции.
- **Связано с**: баг #2
- **Сложность**: высокая
