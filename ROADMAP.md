# Roadmap: Operator Precedence in Math Expressions

Цель: добавить корректный приоритет операций (`*`, `/`, `%` перед `+`, `-`) во все выражения компилятора C--, включая скобки.

---

## Фаза 1 — Константная свёртка (toke.cpp)

Файл: `toke.cpp`

Функции для переработки:

| Функция | Строка | Тип |
|---------|--------|-----|
| `doconstdwordmath()` | 360 | unsigned long |
| `doconstlongmath()` | 502 | signed long |
| `doconstqwordmath()` | ~250 | unsigned long long |
| `doconstfloatmath()` | ~626 | float |
| `doconstdoublemath()` | — | double |

**Что делать:** Каждая сейчас обрабатывает `+`, `-`, `*`, `/` в плоском цикле. Нужно разделить на `term()` (*, /, %) и `expr()` (+, -).

**Риск:** минимальный — чистая арифметика, нет генерации кода.

---

## Фаза 2 — do_e_axmath2 (tokb.cpp, 32/16-bit → EAX/AX)

Файл: `tokb.cpp`, строка ~7187

**Что делать:** Главный путь для `int`/`dword` переменных. Рефакторинг плоского цикла в рекурсивный спуск:

```
factor()  → number | variable | register | (expr)
term()    → factor { (*|/|%) factor }
expr()    → term { (+|-) term }
```

Для промежуточных результатов — push/pop на стек (или двухоперандный `imul reg,reg` / `lea`).

**Потенциальные проблемы:**
- Стек FPU для `float`/`double`
- `expand` (64-bit расширение через DX:AX)
- Оптимизация с `optnumadd` / `RegMulNum`
- Совместимость с `OPT VARCONST`
- `lea`-оптимизация сложения

---

## Фаза 3 — doalmath2 (tokb.cpp, 8-bit → AL)

Файл: `tokb.cpp`, строка ~8459

**Что делать:** По тому же шаблону, что и `do_e_axmath2`, но для 8-битных byte/char.

**Особенности:**
- Только 8-битные регистры (AL, AH, CL...)
- `cbw`/`xor AHAH` для расширения
- Упрощённая кодогенерация по сравнению с 32-bit

---

## Фаза 4 — doregmath_32 (tokb.cpp, 32/16-bit → любой reg)

Файл: `tokb.cpp`, строка ~11917

**Что делать:** Выражения, результат которых идёт не в EAX, а в произвольный регистр (например, `eax = ebx + ecx * 2`).

**Сложность:** Промежуточные результаты могут требовать временного регистра — нужен алгоритм выбора регистра (или стек).

---

## Фаза 5 — dobegmath (tokb.cpp, 8-bit → любой beg-регистр)

Файл: `tokb.cpp`, строка ~12627

**Что делать:** Аналог `doregmath_32` для 8-битных байтовых регистров (AL, CL, DL, BL).

---

## Фаза 6 — Скобки ()

**Что делать:** Во всех 4 фазах выше на уровне `factor()` добавить поддержку `(` `)`.

```
factor() → number | variable | ( expr )
```

При `(` — запомнить позицию, вызвать `expr()` рекурсивно, проверить `)`.

---

## Фаза 7 — Тестирование

- Сборка компилятора (`Makefile.win32` / `Makefile.lin32`)
- Прогон существующих тестов
- Тестовые выражения: `a * b + c`, `a + b * c`, `a + b * c / d`, `a * (b + c)` (после фазы 6)
- Проверка констант: `2 + 3 * 4` → 14 (не 20)
- Регрессия: убедиться, что старый код без смешанных операторов не сломался

---

## Итоговая схема вызовов

```
expr()              ← +, -
  term()            ← *, /, %
    factor()        ← number, variable, (expr)
```

Каждая функция при необходимости push'ит промежуточный результат на стек, а после вычисления pop'ит обратно.
